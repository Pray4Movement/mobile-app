## üìã Compliance Checklist

### Before Submission

#### App Store Connect Setup
- [ ] Create App Store Connect account and app listing
- [ ] Complete app metadata (name, description, keywords, category)
- [ ] Upload app screenshots for all required device sizes
- [ ] Upload app icon (1024x1024px)
- [ ] Set app age rating (complete questionnaire)
- [ ] Add privacy policy URL (REQUIRED)
- [ ] Provide demo account credentials if app requires login
- [ ] Complete App Review Information section
- [ ] Set up pricing and availability

#### Privacy Requirements
- [ ] Create comprehensive privacy policy covering:
  - Data collection (campaign subscriptions, prayer schedules, preferences, completion status, session tracking)
  - Data storage (local storage, caching)
  - Data usage (how data is used within app)
  - Data sharing (if any third-party services are used)
  - User rights (data access, deletion)
- [ ] Host privacy policy at publicly accessible URL
- [ ] Add privacy policy URL to App Store Connect
- [ ] Review all third-party SDKs for privacy compliance
- [ ] Ensure no data collection without user consent

#### Content Requirements
- [ ] Review all prayer content for:
  - Defamatory or discriminatory content
  - Inflammatory religious commentary
  - Inaccurate religious text quotations
  - References to violence or conflict
- [ ] Ensure all content is properly licensed or owned
- [ ] Verify content doesn't violate intellectual property rights

#### Technical Requirements
- [ ] Test app on all supported iOS devices
- [ ] Ensure app targets recent iOS version (iOS 15+ recommended)
- [ ] Test all features for crashes and bugs
- [ ] Verify network error handling and retry logic
- [ ] Test notification functionality
- [ ] Test offline functionality (cached content)
- [ ] Verify deep linking from notifications works correctly
- [ ] Test app with slow/no network connection
- [ ] Ensure app doesn't drain battery excessively
- [ ] Verify app doesn't use excessive memory

#### Developer Code of Conduct
- [ ] Ensure app provides genuine functionality
- [ ] Verify app is not a copy of another app
- [ ] Ensure app metadata is accurate
- [ ] Plan for appropriate responses to App Store reviews
- [ ] Set up review prompt using official API (if adding later)

#### Permissions Justification
- [ ] Notification permission - clearly explain for prayer reminders
- [ ] Network access - required for API calls
- [ ] Storage - for caching content offline

### App Review Information

#### Required Information
- [ ] Provide demo account credentials (if app requires login)
- [ ] Explain any non-obvious features in App Review notes
- [ ] Provide any special hardware or resources needed for review
- [ ] Ensure backend services are live and accessible during review
- [ ] Test that demo account has full access to app features

### Post-Submission

#### Monitoring
- [ ] Monitor App Store Connect for review status
- [ ] Respond promptly to any App Review questions
- [ ] Address any rejection reasons thoroughly
- [ ] Monitor user reviews and respond appropriately
- [ ] Track app quality metrics (crashes, negative reviews)
